# Телеграм бот для сохранения стикеров в стикерпак

Бот автоматически сохраняет все стикеры, отправленные в группу, в общий стикерпак.

## Возможности

- Автоматическое сохранение всех стикеров из группы
- Поддержка статических, анимированных и видео стикеров
- Раздельные паки для каждого типа стикеров
- Автоматическое создание нового пака при переполнении:
  - Статические: 120 стикеров
  - Анимированные: 50 стикеров
  - Видео: 50 стикеров
- Название пака соответствует названию группы + тип
- Отправка ссылки на пак при создании первого стикера каждого типа

## Установка

1. Установите зависимости:
```bash
pip install -r requirements.txt
```

2. Создайте бота через [@BotFather](https://t.me/botfather) и получите токен

3. Установите токен бота:
```bash
export BOT_TOKEN="ваш_токен_бота"
```

Или отредактируйте `config.py` и вставьте токен напрямую.

## Запуск

### Способ 1: Напрямую с переменной окружения
```bash
export BOT_TOKEN="ваш_токен"
python3 bot.py
```

### Способ 2: Через Python скрипт (рекомендуется для NixOS)
```bash
# Создайте файл .env с токеном
echo "BOT_TOKEN=ваш_токен" > .env
# Запустите
./start.py
# или
python3 start.py
```

### Способ 3: Через bash скрипт (Ubuntu/Debian)
```bash
# Создайте файл .env с токеном
echo "BOT_TOKEN=ваш_токен" > .env
# Запустите
./run.sh
```

## Использование

1. Добавьте бота в вашу группу
2. Отправьте любой стикер в группу (статический, анимированный или видео)
3. При первом стикере каждого типа бот создаст отдельный стикерпак и отправит ссылку
4. Все последующие стикеры будут автоматически добавляться в соответствующий пак
5. При заполнении пака автоматически создастся новый:
   - Статические: после 120 стикеров
   - Анимированные: после 50 стикеров
   - Видео: после 50 стикеров

## Структура проекта

- `bot.py` - основной файл бота
- `config.py` - конфигурация (токен, ID владельца, лимиты)
- `packs_state.json` - файл состояния (создается автоматически)
- `requirements.txt` - зависимости Python

## Конфигурация

В файле `config.py`:
- `BOT_TOKEN` - токен бота
- `OWNER_USER_ID` - ID владельца стикерпаков (435959840)
- `ALLOWED_GROUPS` - список ID разрешенных групп (пустой список = все группы)
- `MAX_STICKERS_STATIC` - максимум статических стикеров (120)
- `MAX_STICKERS_ANIMATED` - максимум анимированных стикеров (50)
- `MAX_STICKERS_VIDEO` - максимум видео стикеров (50)

### Ограничение доступа к определенным группам

Чтобы бот работал только в определенных группах, добавьте их ID в `ALLOWED_GROUPS`:

```python
ALLOWED_GROUPS = [
    -5255145507,      # ID первой группы
    -1003463889030,   # ID второй группы
    -1002772620324    # ID третьей группы
]
```

Чтобы узнать ID группы:
1. Добавьте бота в группу
2. Отправьте `/start`
3. Бот покажет ID группы в сообщении об ошибке (если группа не авторизована)

Чтобы разрешить работу во всех группах:
```python
ALLOWED_GROUPS = []  # Пустой список
```

## Особенности

- Каждая группа имеет отдельные паки для каждого типа стикеров
- Название пака формируется из названия группы + суффикс типа:
  - `_s` для статических
  - `_a` для анимированных
  - `_v` для видео
- При переполнении пака создается новый с номером (например: `GroupName_s1`, `GroupName_s2`)
- Состояние паков сохраняется в JSON файле
- Поддерживаются форматы: static (webp), animated (tgs), video (webm)
- **ВАЖНО**: Нельзя смешивать типы стикеров в одном паке (ограничение Telegram API)

## Логирование

Бот ведет подробное логирование всех операций:
- Получение стикеров
- Создание/обновление паков
- Ошибки API
